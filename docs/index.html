One docs hub: learn fast, inspect semantics, run code in-browser.

Vexel is constrained by design so the frontend can model the full graph, execute compile-time paths deeply, and remove dead code before backend emission. Use this page as the entry point to tutorial flow, normative RFC, chaptered specification, backend contracts, and the live playground.

Learning Track (15-20 min)

Tutorial sources live in examples/tutorial/. Each step below opens directly in playground.html.

  1. 1. Quick Add: function + entrypoint baseline — open
  2. 2. Hello + extern: ABI import and side-effect boundary — open
  3. 3. Structs + methods: named types and operators — open
  4. 4. Arrays and shapes: nested arrays and per-element operations — open
  5. 5. std::math lift: bundled math folding and broadcasting — open
  6. 6. std::bits reinterpret: explicit bit reinterpretation — open
  7. 7. Constexpr + DCE: pure folding and dead-symbol removal — open
  8. 8. Prime sieve: stress CTE on nontrivial loops — open
  9. 9. Multi-file modules: import flow across files — open
